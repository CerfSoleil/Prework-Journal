HTML

The Head Element

  • The head element of HTML will detail things that allow the document to be read as HTML and set up for future elements and code seen later.

The Body Element

  • This is currently in he body element of the document.
  • The body element is what you will see in the document when loaded as a web page. It is the only thing the end user should see.

CSS

  • Elements can be made to have the same rules using a comma between selectors, such as "header, footer {}" with the defining style within the brackets.
  • Linking your style sheet to the HTML page requires a reference in the HTML
  • This reference may be set up within the head of the code as "<link rel="stylesheet" href=".RELATIVE/LOCATION.css">"
  • Rules can be applied to specific HTML elements on the CSS code using class selectors where ".REFERENCE" will have the period define the following word as a designation.
  • This then must be added to the HTML sheet in the section designation.

Git

  1. "ssh -T git@github.com" will establish and test your connection to GitHub
  2. "git status" will check what branch you are currently on as well as list untracked changes or additions to the files.
  3. "git checkout -b BRANCH/NAME" will create a new branch and switch to it.
  4. "git add -A" will add the modification in the current working branch to the staging area. "-A" indicates that we want to add all changes.
  5. "git pull origin main" will recieve main or a branch's modifications into the local working area.
  6. "git push origin BRANCH/NAME" will push the branch to GitHub in order for you to share or create a pull request.
  7. "git commit -m "COMMIT MESSAGE" will update the repository with your local changes.
  8. The Pull Request must then be handled on GitHub.
